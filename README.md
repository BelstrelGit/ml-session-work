# ML без учителя: кластеризация сигналов детектора

Сессионный проект по кластеризации сигналов сцинтилляционного детектора без учителя.
Задача: разделить импульсы на физические группы по форме сигнала и подготовить submission для соревнования Kaggle.

## Итоговый результат

- Kaggle competition: [Signal types classification](https://www.kaggle.com/competitions/signal-types-classification)
- Лучший public score: **0.84373**
- Порог максимальной оценки по ТЗ: **0.84**
- Финальный submission: `final_submission/final_submission_084373.csv`

## Основной файл

Для проверки решения используйте ноутбук:

```text
session_work_final_27_05_2026.ipynb
```

Ноутбук содержит полный пайплайн: загрузку данных, EDA, baseline-модели, признаки из документации, композицию методов, финальную доводку PSD-границы, sanity-check submission и интерпретацию кластеров.

## Данные

Исходный файл с сигналами:

```text

```

Файл данных не включается в репозиторий. Его нужно скачать из Kaggle-competition и положить рядом с ноутбуком либо поправить путь `DATA_PATH` в первой ячейке ноутбука.

Формат данных:

- каждая строка соответствует одному сигналу;
- первые 4 значения — метаданные;
- следующие 500 значений — отсчёты формы сигнала;
- в финальной модели метаданные не используются как признаки, но анализируются в EDA.

## Использованные подходы

1. **EDA и предобработка**
   - вычитание нулевой линии по первым 50 отсчётам;
   - оценка положения пика, заряда, амплитуды, уровня шума;
   - анализ корреляций и распределений признаков.

2. **Baseline-модели**
   - KMeans на признаках `PSD_155` и `log(Q_total)`;
   - GMM с подбором `covariance_type` по BIC;
   - IsolationForest для выделения выбросов и KMeans для разбиения оставшихся сигналов.

3. **Методы из документации**
   - PSD (Pulse Shape Discrimination) через отношение интегралов в tail/full gate;
   - время высвечивания через аппроксимацию хвоста экспонентой;
   - PCA по площади и амплитуде сигнала;
   - композиция голосов PSD, tau и PCA.

4. **Финальная доводка**
   - ранжирование пограничных PSD-кандидатов;
   - точечный перевод части объектов между двумя основными классами;
   - финальный sanity-check формата CSV.

## Интерпретация результата

Основное разделение соответствует различию формы импульса:

- низкий PSD — более быстрое затухание, gamma-like сигналы;
- высокий PSD — более длинный хвост, neutron-like сигналы;
- class 2 рассматривался как класс аномальных или неоднозначных сигналов, но в финальном submission явное выделение class 2 ухудшало public score.

Точное соответствие номеров кластеров физическим классам зависит от перестановки меток Kaggle, поэтому в ноутбуке отдельно проверяется финальная перестановка и формат submission.

## Ограничения

- Финальная PSD-доводка калибрована по public leaderboard, поэтому возможен риск переобучения под public split.
- Метаданные первых 4 колонок не используются в финальной модели: EDA показал, что часть метаданных сильно дублирует заряд сигнала.
- Дальнейшее улучшение возможно через более аккуратное выравнивание сигналов по пику и независимую валидацию границы PSD.
